FTC Limelight Training

For Middle School Robotics Students

Goal: Use Limelight 3A vision data in Java to align faster and improve autonomous accuracy.

1. What is Limelight 3A?

Limelight 3A is a vision camera used in FTC to detect targets and provide useful values like horizontal offset, vertical offset, and target visibility.

  • Target tracking: Detect tags and objects.
  • Alignment support: Help drivers line up faster in TeleOp.
  • Autonomous support: Improve consistency when approaching scoring locations.

2. Key Limelight Concepts

2.1 Valid Target

Always check whether Limelight currently sees a valid target before applying corrections.

2.2 Offsets

  • tx: horizontal offset (left/right error)
  • ty: vertical offset (up/down angle error)

2.3 Pipeline

A pipeline defines what vision processing mode Limelight is using. Teams often create different pipelines for AprilTags and game elements.

3. Basic Java Workflow

// 1) Read Limelight result values
// 2) Check if target is valid
// 3) Use tx to compute turn correction
// 4) Send correction to drivetrain motors

Tip: Keep your turn correction capped so alignment stays smooth and controllable.

4. Simple Alignment Example (Concept)

double error = 0 - tx;
double turnPower = kP * error;
turnPower = Math.max(-0.35, Math.min(0.35, turnPower));

leftDrive.setPower(-turnPower);
rightDrive.setPower(turnPower);

if (Math.abs(error) < 1.0) {
    leftDrive.setPower(0);
    rightDrive.setPower(0);
}

5. Practice Activities

  • Run a valid-target check and show status in telemetry.
  • Display tx and ty values every loop.
  • Tune kP so alignment is fast but not shaky.
  • Align to a target and stop within ±1.0 degrees of horizontal error.
